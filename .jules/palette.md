## 2024-05-15 - Icon-only buttons and unassociated textareas need explicit labels
**Learning:** In reusable UI components like `ScriptEditor.tsx`, icon-only utility buttons (e.g., expand/minimize, reset) and standalone inputs like `<textarea>` often lack textual context, making them inaccessible to screen readers. Relying solely on `title` or placeholder text is insufficient for robust accessibility, and keyboard-only users need clear focus rings.
**Action:** Always add explicit `aria-label` attributes to icon-only buttons describing their action (dynamically if stateful), and ensure `<textarea>` or `<input>` elements without `<label>` tags have descriptive `aria-label`s. Also add explicit `focus-visible:ring-2 focus-visible:ring-maya-500 focus:outline-none` classes to icon-only buttons so keyboard focus is visually distinct.
